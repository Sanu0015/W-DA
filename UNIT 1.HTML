History of www

  • The World Wide Web (WWW) was invented by Tim Berners-Lee, a British scientist, in 1989 while working at CERN. The Web was originally developed to meet the demand for automated information-sharing between scientists in CERN, which is an extensive community of scientists from over 100 countries. By the end of 1990, the first Web server and browser were up and running at CERN, and the first Web page address was. The development of the WWW was further accelerated by the release of the Mosaic browser in 1993, which allowed users to use "point-and-click" graphical manipulations. The early web community produced revolutionary ideas such as decentralization and open access, leading to the founding of the World Wide Web Consortium (W3C) in 1994. This decision sparked a global wave of creativity, collaboration, and innovation. In 2014, almost two in five people around the world were using the Web, and in the same year, the companies developing new web standards committed to a Royalty Free Policy for their work.

Evolution of Html

  • HTML, the markup language used to create web pages, has evolved through various versions since its inception. Here is a brief overview of its evolution:
    - HTML 1.0 (1993): The first version of HTML was released in 1993, providing the basic structure and elements for creating web pages. It consisted of 18 tags and was intended to share information readable and accessible via web browsers.
    - HTML 2.0 (1995): This version contained all the features of HTML 1.0 along with a few additional features. It remained the standard markup language for designing and creating websites until 1997.
    - HTML 3.0 (1997): Introduced a fresh draft on HTML with improved new features, providing more powerful characteristics for webmasters in designing websites. However, the powerful features slowed down the browser in applying further improvements.
    - HTML 4.01 (1999): This was a widely used and successful version of HTML, introducing advancements such as Cascading Style Sheets (CSS) for improved control over page styling, enhanced support for scripting, and multimedia elements.
    - HTML 5.0 (2014): HTML 5 can be considered an extended version of HTML 4.01, which was published in 2012. It marked a significant milestone in web development, introducing a wide range of new elements and attributes, improved multimedia support, and better compatibility with mobile devices.
    HTML has continued to evolve, and its various versions have unfurled new possibilities, pushing the boundaries of creativity, connectivity, and user experience.

Different Html Versions

  • HTML (Hypertext Markup Language) is an evolving language with different versions supporting different features, and each new version is given a version number. The first version of HTML did not have a version number and was just called HTML. The major versions of HTML include HTML 2.0, released in 1995, which was the standard for website design until January 1997 and defined many core HTML features for the first time. HTML 3.2 was finalized by the W3C in early 1997 and included support for creating tables and expanded options for form elements. HTML 4.01 is the current official standard, released in December 1999, and added support for style sheets and scripting ability for multimedia elements. HTML5 is still a work in progress, and all major browsers support many of the new HTML5 elements and APIs. Its core aim has been to improve the language with support for the latest multimedia while keeping it easily readable by humans and consistently understood by computers and devices.

Why Html5

  • HTML5, the fifth version of the Hypertext Markup Language, was introduced to address the limitations of its predecessors and to provide enhanced support for the latest multimedia, improved semantics, and a better user experience. Some of the key features of HTML5 include:
    1.Semantic Elements: HTML5 introduced new semantic elements like header, footer, and section to provide a better structure and meaning to web content.
    2.Audio and Video Support: HTML5 includes native support for embedding audio and video content without the need for third-party plugins like Adobe Flash.
    3.Canvas Elements: It provides a built-in drawing surface using JavaScript, allowing for the creation of dynamic graphics and visualizations directly in the browser.
    4.Local Storage: HTML5 offers improved storage options, such as application cache, web SQL database, and web storage, reducing the reliance on browser temporary storage.
    5.Responsive Images: HTML5 introduced new features for creating responsive images, ensuring proper rendering on different devices and screen sizes.
    6.Web Workers: It allows web applications to run scripts in the background, enabling multi-threading in web applications for better performance.
    7.Drag and Drop API: HTML5 provides native support for drag and drop functionality, making it easier to implement interactive user interfaces.
    8.Form Enhancements: HTML5 introduced new form input types, attributes, and validation features, improving the user experience and making form handling more robust.
    9.Web Sockets: It enables real-time communication between a client and a server, allowing for bi-directional communication.
    10.Geolocation API: HTML5 allows websites to request the physical location of users, enabling location-aware web applications.
    These features have made HTML5 a vital standard for web development, offering improved capabilities, better user experience, and reduced reliance on third-party plugins. It is widely supported by modern browsers, making it the preferred choice for building modern websites and web applications.

HTML4 vs HTML5

  • HTML (Hypertext Markup Language) is the standard markup language for creating and structuring web pages. HTML4 and HTML5 are two different versions of the HTML specification, with HTML5 being the latest and more modern version. Here are some key differences between HTML4 and HTML5:
    1. DOCTYPE Declaration:
    - HTML4: Requires a strict document type declaration at the beginning of the HTML document.
    - HTML5:Simplifies the doctype declaration to `<!DOCTYPE html>`, making it more straightforward.
    2. Syntax:
    - HTML4: Has a more rigid syntax and is less forgiving of errors. It may not display properly if there are minor mistakes in the code.
    - HTML5: Has a more lenient syntax and is designed to be more forgiving. It can handle minor errors more gracefully, making it easier for developers.
    3. New Elements:
    - HTML4: Lacks several modern elements that are available in HTML5, such as <header>, <footer>, <nav>, <article>, <section>, <aside>, <figure>, and <figcaption>.
    - HTML5: Introduces new semantic elements that provide better structure and meaning to the content of a web page.
    4. Multimedia:
    - HTML4: Relies on plugins like Adobe Flash for multimedia content.
    - HTML5: Supports native multimedia elements like `<audio>` and `<video>`, reducing the dependence on third-party plugins.
    5. Graphics:
    - HTML4: Requires the use of external technologies like SVG or Canvas for advanced graphics.
    - HTML5: Introduces the `<svg>` element for scalable vector graphics and the `<canvas>` element for dynamic rendering of graphics and images.
    6. Form Elements:
    - HTML4: Limited form controls and validation options.
    - HTML5: Introduces new form input types (`<input type="email">`, `<input type="date">`, etc.) and attributes for more advanced form handling and validation.
    7. APIs:
    - HTML4: Lacks modern APIs for offline web applications, geolocation, drag-and-drop, and more.
    - HTML5: Includes various new APIs, such as the Geolocation API, Web Storage API, Web Workers API, and the Drag and Drop API.
    8. Compatibility:
    - HTML4: Designed in an era when the web was less dynamic and interactive.
    - HTML5: Adapted to the current needs of the web, with a focus on multimedia, interactivity, and improved semantics.
    In summary, HTML5 is a more feature-rich and flexible version of HTML, designed to meet the demands of modern web development. It introduces new elements, attributes, and APIs, making it easier for developers to create interactive and multimedia-rich web applications.

Browser support

  • The browser support for various web technologies and features can be checked using different resources. One such resource is "Can I use" (caniuse.com), which provides support tables for HTML5, CSS3, and other web technologies. Additionally, Google provides information on supported browsers for signing in and using its services, emphasizing the importance of using a supported browser and enabling JavaScript for security reasons. Furthermore, developers can find information on supported browsers and platforms for Google sign-in and other features on the Google Developers website. For specific web technologies like CSS, the W3Schools website offers a CSS Browser Support Reference, detailing the support for CSS properties in different browsers. Moreover, Mozilla's Developer Network (MDN) provides information on browser support for various JavaScript APIs. These resources can help developers ensure the compatibility of their web content and applications across different browsers and platforms.

Backward compatibility

  • Backward compatibility in the context of HTML5 refers to the ability of the language to support existing web content and applications, ensuring that they continue to function properly even as new features are introduced. HTML5 is designed to be as backward compatible as possible with existing web browsers. New features build on existing ones and allow for the provision of fallback content for older browsers. This means that while HTML5 introduces new capabilities, it also ensures that existing web content and applications remain functional. However, it's important to note that even HTML5-aware web browsers do not support the HTML5 specification perfectly, so providing some form of backward compatibility with all HTML5 features is necessary to avoid alienating users based on their choice of web browser.
